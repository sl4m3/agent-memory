# Process Invariants (v1)

Этот документ описывает инварианты процесса — правила, которые должны соблюдаться при изменении состояния памяти.

## PI1: Review Window (Защита от поспешности)
- **Правило**: `Proposal` не может быть принят (`accept_proposal`) ранее, чем через **1 час** после его создания, если источником является `reflection_engine`.
- **Исключение**: Прямая команда человека (TrustBoundary.HUMAN_ONLY).
- **Зачем**: Предотвратить циклы самоподтверждения, когда агент мгновенно принимает свои же гипотезы.

## PI2: Evidence Threshold (Минимальное обоснование)
- **Правило**: `ReflectionEngine` не имеет права создавать `Proposal`, если у него нет как минимум **3 эпизодических улик** (evidence events).
- **Зачем**: Избежать индукции на основе единичных случаев.

## PI3: Immutable Traceability
- **Правило**: При переходе `Proposal -> Decision`, исходный ID пропозала ОБЯЗАН быть записан в метаданные решения.
- **Зачем**: Сохранение неразрывной связи между гипотезой и истиной.

## PI4: Role-Based Transition
- **Правило**: Агент может инициировать `supersede_decision` только для таргетов, которые он сам же создал. Изменение фундаментальных правил системы (созданных человеком) требует внешнего ResolutionIntent.
